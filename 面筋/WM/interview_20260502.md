## 日期：20260502

### WM (WorldModel/规划) 领域

**论文：《GuideFlow: Constraint-Guided Flow Matching for Planning in End-to-End Autonomous Driving》**

1. [基础概念题] 请解释 Flow Matching（流匹配）与传统 Diffusion Model 在生成轨迹时的核心区别。为什么 GuideFlow 选择 Flow Matching 而不是 Diffusion 作为生成式规划的基础？
   - **答案/思考提示**：Diffusion 从噪声逐步去噪（p_θ(x_t|c) → p_θ(x_{t-1}|c)），步数多、推理延迟大，约束难以自然作用到逐步去噪的内部状态。Flow Matching 直接学习连续向量场 v_t(x|c)，将简单先验分布 p_0 映射到目标轨迹分布 p_1，具有连续性强（采样过程有轨迹演化解释）和可干预性强（可改速度场、流状态、能量层面）的优势，特别适合「约束嵌入式生成」。

2. [深度思考题] GuideFlow 提出了三层约束机制：CVF（约束速度场）、CF（约束流状态）、RFE（奖励/规则感知流能量优化）。请分析这三层机制各自的优缺点，以及为什么 CF + RFE 组合效果最优？
   - **答案/思考提示**：
     - **CVF**：每一步对速度场做修正，是「连续纠偏」。优点是早期干预、防止轨迹开始就走错方向；缺点是修得太强可能破坏概率路径平滑性，更像连续纠偏而非最小必要干预。
     - **CF**：在关键时刻 t_k 做一次状态替换，是「关键节点纠偏」。优点是减少对自然生成质量的损害，让后续轨迹从更合理的状态继续往前流；缺点是需要选择合适的干预时机。
     - **RFE**：在训练时塑形能量地形，使模型偏爱满足约束的区域。优点是OOD场景泛化潜力更强，约束成为模型内部偏好而非后验筛选器；缺点是训练复杂度增加。
     - **CF + RFE 最优原因**：CF 处理生成过程中的中后期局部修正，RFE 从全局塑造目标分布的低能量结构，二者互补——CF 负责「事中干预」，RFE 负责「事前塑形」。

3. [深度思考题] GuideFlow 论文强调「约束不能只在终点做筛选，而应被嵌入生成动力学本身」。这个观点对于端到端自动驾驶规划的意义是什么？对比「生成后裁判（Scorer）」和「生成前教练（RFE）」这两种约束处理方式的本质区别。
   - **答案/思考提示**：传统 scorer 是生成后评估轨迹，属于「后验筛选器」，只告诉模型哪个轨迹好，但不教模型如何生成好的轨迹。RFE 将约束内化为能量地形的一部分，在训练时就塑形模型对约束区域的偏好，属于「生成前教练」。本质区别在于：Scorer 优化的是「选择」，RFE 优化的是「生成能力本身」。对于自动驾驶这种对安全性要求极高的场景，内生约束比后验筛选更具鲁棒性，因为 OOD 场景下 scorer 可能失效，而内化的约束偏好仍能发挥作用。

4. [实践应用题] 假设你要将 GuideFlow 的思想应用到机器人操纵（manipulation）任务中，需要对三层约束机制进行适配。请说明：
   - 如何定义机器人操纵任务中的 CVF、CF、RFE？
   - 如果某些约束（如末端执行器姿态）比其他约束（如关节速度限制）更难满足，应该如何调整三层机制的权重或介入时机？
   - **答案/思考提示**：
     - **CVF**：可定义为关节速度场修正，当末端执行器轨迹偏离安全区域时，通过修正速度场方向将其「折回」可行方向
     - **CF**：在关键时间步（如接近目标物体、进入障碍物附近）做状态重置，将末端执行器投影到可行流形
     - **RFE**：将碰撞约束、姿态约束、任务目标等建模为能量函数，在训练时让模型学会「本能地」回避低能量（高约束违反）的配置
     - **权重调整**：对于难以满足的约束，可在 RFE 层面提高能量惩罚系数 λ，或在 CF 层面选择更早的介入时间点 t_k，让修正有更多时间步发挥作用

5. [优缺点对比题] 与 Uni-World VLA、Being-H0.7、π0.7 等世界模型规划方法相比，GuideFlow 的核心优势和局限性是什么？在什么场景下你会优先选择 GuideFlow 而不是这些方法？
   - **答案/思考提示**：
     - **优势**：GuideFlow 专注于「约束嵌入式多模态生成」，在多样化轨迹生成的同时保证安全/规则约束，方法论上更聚焦于规划的可控性；Uni-World VLA 等侧重于「统一架构」和「泛化能力」，GuideFlow 侧重于「生成质量与安全性的统一」
     - **局限性**：GuideFlow 主要仍是轨迹层生成，尚未完全扩展到长时程 latent world planning；采样效率仍是现实部署的瓶颈
     - **优先选择场景**：需要多模态轨迹候选但又必须满足严格安全约束的场景（如高速公路驾驶、复杂城市道路）；当需要可调节的「驾驶风格」控制时（通过调整 guidance 强度 w）；当希望在不需要外部 scorer 的情况下获得基础安全性保证时

### RL 领域

**通用思考题（无当日新论文）**

1. [基础概念题] PPO（Proximal Policy Optimization）算法中，Clipped Surrogate Objective 的设计是为了解决什么问题？请解释 KL 散度约束与 Clip 机制的区别与联系。
   - **答案/思考提示**：PPO 的 Clip 机制旨在解决策略梯度方法中步长过大的问题——如果一次更新让策略变化太大，训练会不稳定甚至崩溃。具体通过将策略比率 r_t(θ) = π_θ(a_t|s_t) / π_{θ_old}(a_t|s_t) 限制在 [1-ε, 1+ε] 范围内，防止极端策略更新。KL 散度约束通过直接限制新旧策略的 KL 散度上界来控制步长，PPO-Penalty 用拉格朗日项加入目标函数，PPO-Clip 则通过 Clip 操作隐式地实现了类似效果。相比之下，Clip 更简单高效，不需要显式计算 KL 散度。

2. [深度思考题] 在 RLHF（Reinforcement Learning from Human Feedback）中，Reward Model 的质量直接决定了最终策略的效果。为什么用 Reward Model 而非直接用人类偏好训练策略？Reward Model 存在哪些潜在问题，以及如何缓解？
   - **答案/思考提示**：Reward Model 本质上是一个偏好分布的压缩表示，允许在任意状态下查询任意动作的相对偏好，而无需每次都请人标注。具体优势包括：可复用（一次训练多次使用）、可微（支持梯度反传）、可泛化（学习到偏好背后的隐含规律）。潜在问题包括：①Reward hacking（策略学会「骗过」RM 而非真正完成目标）；②RM 分布外泛化差（对未见过的状态-动作对预测不准）；③RM 本身可能存在偏见。缓解方法包括：iverse reward modeling、KL 约束限制策略偏离 SFT 模型、对抗性训练等。

3. [实践应用题] 你正在开发一个四足机器人的 locomotion 控制器，已有一个基于 PPO 训练的策略，但发现机器人在不平整地面上容易摔倒。请列出可能的原因以及对应的算法改进方向。
   - **答案/思考提示**：可能原因包括：①训练分布不够多样（只用了平整地面）；②奖励函数中对平衡/稳定性的激励不足；③观测信息缺少触觉或地面不平整度等关键信号。改进方向：①Domain Randomization：在训练时随机引入地面高度变化、摩擦系数变化等；②奖励塑形：增加对躯干姿态稳定性、支撑多边形面积、摆动腿协调性等的奖励；③课程学习：从平整地面逐步过渡到复杂地形；④加入特权信息（如地形先验）作为 critic 的额外输入。

4. [优缺点对比题] 比较 Model-based RL（如 Dreamer、World Models）和 Model-free RL（如 PPO、SAC）在 sample efficiency 和 asymptotic performance 上的trade-off。为什么机器人控制任务通常更关注 sample efficiency？
   - **答案/思考提示**：Model-based RL 通过学习世界模型来模拟环境交互，通常 sample efficiency 更高（因为可以在模型内部进行大量想象 rollout），但 asymptotic performance 受限于模型误差累积；Model-free RL 直接从真实交互学习，asymptotic performance 上限更高，但需要大量真实样本。机器人控制任务通常 physical sample efficiency 极低（真实机器人交互成本高、磨损大），且任务边界相对清晰（模型误差更容易控制），因此 model-based 方法更受青睐。但当任务过于复杂、状态空间巨大或奖励函数设计困难时，model-free 方法的灵活性和最终性能上限可能更有优势。
