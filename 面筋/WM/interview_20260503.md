## 日期：20260503

### WM (WorldModel/规划) 领域

**通用思考题（当日无新论文）**

1. [基础概念题] 世界模型（World Model）在强化学习中通常扮演什么角色？对比 Dreamer 系列、DreamerAD、Fast-WAM、Being-H0.7 等方法，它们在学到的世界模型类型（deterministic vs stochastic、latent vs pixel-level）上有什么区别？
   - **答案/思考提示**：世界模型的核心作用是通过无模型预测环境未来状态，让 agent 可以在模型内部进行大量「想象 rollout」，从而提高 sample efficiency。学到的世界模型类型决定了：
     - **Deterministic vs Stochastic**：Dreamer 是 deterministic（确定性）世界模型，状态转移是确定函数；Stochastic 模型（如 DreamerAD 可能引入随机性）会建模状态转移的分布不确定性。
     - **Latent vs Pixel-level**：Dreamer 系列（如 DreamerV2/V3）在 latent space（隐空间）进行预测，输入是 compact 状态表征而非原始像素；Pixel-level 模型（如早期 World Models）直接在像素空间预测，想象 rollout 的计算成本高但保真度更高。
     - **Fast-WAM**：专注 World Action Model 规划加速，可能在预测效率上有特殊设计。
     - **Being-H0.7**：端到端 JEPA 世界模型，可能采用了 ViT 类架构进行掩码预测学习。
     - **DreamerAD**：专门针对自动驾驶场景的世界模型，需要建模道路、车辆、行人等复杂动态。

2. [深度思考题] LeWorldModel 提出了「首个端到端 JEPA 世界模型」，JEPA（Joint Embedding Predictive Architecture）与传统的 Autoencoder 或 VAE 架构在「学习表征」层面有什么本质区别？为什么说 JEPA 更适合学习用于规划的表征？
   - **答案/思考提示**：传统 Autoencoder/VAE 通过重建像素级输入来学习表征，目标是「重建原始信号」，但这会导致模型学习到细节（ texture、噪声）而非语义信息。JEPA 通过掩码重建在表征空间进行预测，目标是「预测掩码部分的表征」，这迫使模型学习高层次的语义结构而不是底层的像素细节。
     - **为什么 JEPA 适合规划**：①学到的表征更抽象、更语义化，适合迁移到不同任务；②预测的是表征空间中的状态，而不是像素空间，这意味着学到的动态更接近「物理规律」而非「视觉规律」；③对于未见过的场景，语义级预测比像素级预测更鲁棒。
     - **JEPA vs MAE**：MAE 重建像素，JEPA 重建表征；MAE 仍然是 reconstructive，JEPA 是 predictive。

3. [深度思考题] 在自动驾驶规划中，为什么需要「世界模型」而不是直接用感知-规划端到端模型？世界模型如何帮助解决长尾问题（long-tail scenarios）？与传统基于规则的规划器相比，基于世界模型的规划有什么优势？
   - **答案/思考提示**：
     - **为什么需要世界模型**：端到端规划器直接从当前感知输出控制指令，缺乏对未来的预测能力；世界模型能够「想象」当前决策会导致什么样的未来状态，从而进行多步 lookahead 规划，更好地处理紧急情况。
     - **解决长尾问题**：世界模型可以在虚拟环境中合成罕见场景（雨雪天、交通事故等），让规划器在安全环境中学习应对；也可以通过模型预测识别当前场景是否属于长尾分布，如果是则切换到保守策略。
     - **相比规则规划器的优势**：规则规划器依赖人工设计的成本函数和约束，难以覆盖所有场景；世界模型能学习更复杂的驾驶策略，成本函数可以通过数据自动学习而非手工设计。

4. [实践应用题] 假设你要为物流机器人（室内仓储场景）设计世界模型+规划的架构，需要：
   - 支持动态障碍物（其他机器人、人员）的建模
   - 支持多机器人协作任务（如「机器人A去取货架X，机器人B去取货架Y」）
   - 推理时间 < 50ms（实时控制要求）
   
   请说明：①世界模型应该建模哪些关键元素（状态空间、动作空间、时间尺度）？②多机器人协作如何融入世界模型框架？③如何在保证实时性前提下利用世界模型做规划？
   - **答案/思考提示**：
     - **世界模型设计**：状态空间应包含自身 pose、其他智能体的 pose、货架/目标物体位置、地图的语义分割；动作空间是底层的电机控制或 mid-level 的导航指令；时间尺度上，短期（<1s）需要精细避障，长期（>5s）需要任务级规划。可考虑分层世界模型：low-level 模型高频预测物理碰撞，high-level 模型低频预测任务进度。
     - **多机器人协作**：每个机器人都可以有自己的世界模型，也可以有一个共享的中心世界模型建模所有机器人的联合状态；协作意图通过动作空间中的「协作信号」或通讯信道传递。也可使用 game-theoretic 框架建模多智能体交互。
     - **实时性保证**：①世界模型在隐空间预测而非像素空间，计算量小；②使用 MPC（模型预测控制）做短时域规划（10-20步），长时域用启发式；③世界模型可以提前预热（pre-compute），只在新场景时更新；④使用 action chunking 一次输出多步动作减少推理频率。

5. [优缺点对比题] 对比 Being-H0.7、Fast-WAM、π0.7 这三个工作：
   - 它们在「动作执行方式」（开环 vs 闭环、模仿学习 vs 强化学习）和「世界模型用途」（规划 vs 辅助训练）上的分工有什么不同？
   - 如果你要做「家庭服务机器人长时程任务」（如「整理房间」），你会优先选择哪个方法？为什么？
   - **答案/思考提示**：
     - **Being-H0.7**：端到端 JEPA 世界模型 + 动作生成，主要用于规划（想象未来场景），动作执行可能是开环的（直接执行想象的轨迹）。
     - **Fast-WAM**：World Action Model，核心是加速规划过程，可能在推理速度上有特殊优化，适合需要快速响应的场景。
     - **π0.7**：使用 diffusion action experts，更强调闭环控制能力，通过多层 action experts 实现精细操控，世界模型更多用于辅助训练（imagination）而非直接规划。
     - **家庭服务机器人选择**：家庭场景任务复杂、长时程、需要精细操控（抓取、放置），且环境动态变化（物体位置不固定），更适合 π0.7 的闭环控制 + 多层级 action experts 方案；如果任务更多是导航+简单操作，Being-H0.7 的世界模型规划可能更高效。实际系统可能需要组合：π0.7 的精细操控 + 世界模型的 long-horizon 规划。
