### WM 领域（World Model / World Action Model）

**论文1：《Motus：Unified World Model / World Action Model》**

1. **[基础概念题]** Motus 提出的 Unified World Model 与传统 World Model 的本质区别是什么？传统 WM 只建模 $p(o_{t+1}|o_t, a_t)$ 这一个条件分布，而 Unified WM 同时建模哪四种概率分布？请列举并说明它们各自的物理意义。
   - **答案/思考提示**：① $p(o_{t+k}|o_t)$ VGM（视频生成）② $p(a_{t+k}|o_t, o_{t+k})$ IDM（逆动力学）③ $p(a_{t+k}|o_t, a_t)$ Policy/VLA ④ $p(o_{t+1}|o_t, a_t)$ 传统WM（dynamics）。本质区别是建模对象从"一个条件分布"扩展到"完整的联合分布"。

2. **[深度思考题]** 为什么 Motus 选择在视频生成框架中引入"Latent Action"（潜在动作）？不使用真实动作标签有什么优势？这与具身智能的数据稀缺问题有什么关系？
   - **答案/思考提示**：① 很多数据本身没有action label或噪声大 ② 不同机器人本体的action space不一致 ③ Latent action提供统一表示，可以对齐不同来源的动作信号 ④ 使得模型可以吃进互联网视频、图文数据等海量无标注数据

3. **[深度思考题]** Motus 论文提到多任务 scaling 的有趣现象：任务越多，Motus 的平均成功率反而提升，而 π0.5 反而下降。请从"world knowledge vs task-specific knowledge"的角度解释这个现象。这对通用机器人学习有什么启示？
   - **答案/思考提示**：Motus 学到的是任务间共享的"世界知识"（物理规律、因果结构），而 π0.5 只是在孤立地学每个任务。启示：构建世界模型比单纯做 task-specific policy 更有利于泛化，任务越多可以学到更general的世界规律。

4. **[实践应用题]** 假设你要设计一个机械臂控制系统，可以选择三条路径中的任意一条：① IDM路线（VGM → IDM → Policy）② World Model路线（WM → RL → Policy）③ Unified路线（Video ↔ Action联合建模）。请分析在什么场景下你会选择每条路线，并说明理由。
   - **答案/思考提示**：① 当有海量无标注视频数据但缺动作标签时选IDM ② 当需要精确控制和高样本效率时选WM+RL ③ 当数据来源多样且追求通用性时选Unified。Unified适合数据丰富但标注稀缺的情况。

5. **[优缺点对比题]** 将 Motus 的 Unified WM 与 Yann LeCun 的 JEPA 架构进行对比：两者都旨在建模世界，但核心思想有什么根本差异？JEPA 主张"不画世界，直接学习抽象结构"，Motus 却在做视频生成，这两种路线哪个更接近真正的世界模型？为什么？
   - **答案/思考提示**：JEPA在表征空间做预测（避免像素级误差累积），Motus在像素空间生成视频。JEPA更理论优雅，但Motus的"生成即理解"路线在实践中更容易scale。两者并不互斥——理想的世界模型可能需要两者的结合。

**论文2：《世界模型全面解析：定义、路线、实践与AGI》**

1. **[基础概念题]** 视频将世界模型的核心能力分为三个维度：观察（Observe）、预测（Predict）、在内部世界中学习行动（Learn Action within Internal Model）。请分别解释这三个能力在机器人控制场景中是如何协作的？能否给出一个具体例子？
   - **答案/思考提示**：观察理解当前状态→预测动作后果→在内部模型中规划行动。例如机器人抓取物体：观察物体位置和形状（Observe），预测如果从侧面抓会滑落（Predict），在内部模拟不同抓取角度的效果后选择最优方案执行（Learn Action）。

2. **[基础概念题]** 视频介绍了四条主要技术路线：视频生成（Sora/Genie 3）、3D空间智能（World Labs）、智能体训练（SIMA）、JEPA抽象预测。请简述每条路线的核心方法及其代表性工作。
   - **答案/思考提示**：① Sora"播放世界"——DiT架构生成视频 ② Genie 3"探索世界"——视频生成+动作规划 ③ World Labs李飞飞——NeRF/3DGS构建3D可交互场景 ④ SIMA——游戏环境训练通用Agent ⑤ JEPA——在表征空间做预测而非像素空间

3. **[深度思考题]** Yann LeCun 批评当前LLM发展路线是"路线错了"，理由是 Scaling Law 和 next token prediction 无法实现真正的通用智能。请分析 LeCun 的批评是否有道理？他的 JEPA 路线能否真正解决这一问题？
   - **答案/思考提示**：LeCun的批评有一定道理——LLM确实缺乏物理世界直觉。但JEPA也有局限：抽象表征可能丢失细粒度物理信息，且纯自监督学习缺乏任务目标引导。可能的解决方向是结合世界模型的预测能力 + LLM的语义理解能力。

4. **[实践应用题]** 视频提到世界模型在具身智能领域可以"降低仿真到现实的迁移门槛"（Sim-to-Real Gap）。请设计一个具体方案，说明如何利用世界模型来生成多样化的仿真训练场景，从而提升机器人在真实环境中的泛化能力。
   - **答案/思考提示**：① 用世界模型生成多样化场景变体（不同光照、遮挡、物体排列）② 在生成的仿真环境中做RL训练 ③ 通过IDM或逆动力学将仿真中学到的技能迁移到真实机器人 ④ 关键是世界模型需要足够精准地预测物理交互结果

5. **[优缺点对比题]** 对比 Sora 路线（视频生成）和 JEPA 路线（表征预测）在以下维度上的优劣：① 物理规律理解 ② 计算效率 ③ 长时序预测能力 ④ 误差累积问题。哪个路线更适合作为机器人控制的世界模型？为什么？
   - **答案/思考提示**：Sora在物理规律理解上依赖涌现（不可控），JEPA更显式但可能丢失细节。计算效率JEPA更优（表征空间vs像素空间）。长时序预测JEPA无误差累积优势明显。机器人控制需要精确和可解释性，JEPA路线更合适，但理想方案可能需要结合两者。

### RL 领域

**注：今日无新 RL 论文，附通用思考题**

1. **[基础概念题]** 在 On-Policy 强化学习算法（如 PPO）中，为什么要使用 Importance Sampling？它解决了什么问题，又带来了什么新问题？
   - **答案/思考提示**：解决策略梯度中"使用新策略采样的数据无法直接估计旧策略期望"的问题。但IS会导致高方差——当新旧策略差异大时，权重可能极端大或小。PPO通过clipped probability ratio来限制这种偏差。

2. **[深度思考题]** Model-based RL（如 Dreamer、World Model）相比 Model-free RL 在样本效率上的优势从何而来？请从"想象力增强"的角度解释。
   - **答案/思考提示**：Model-based RL 学到一个环境模型（世界模型），可以在隐空间中进行"想象式规划"——不需真实与环境交互就能预测动作效果。这大幅减少了真实环境交互次数，特别适合机器人等领域数据稀缺的场景。

3. **[实践应用题]** 如果你需要在有限机器人数据下训练一个机械臂抓取策略，你会选择 Model-based 还是 Model-free 方法？为什么？如果有海量互联网视频数据呢？
   - **答案/思考提示**：有限机器人数据 → Model-based（Dreamer/WM）利用想象力弥补数据不足。海量互联网视频 → 可以考虑 Video-conditioned Policy（如 IDM 路线）或预训练视觉表征。关键是数据的形态和任务的相关性。
