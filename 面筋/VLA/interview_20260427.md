# 面试题整理

## 日期：20260427

### VLA 领域

**论文：《VLA-JEPA: Vision-Language-Action Models with JEPA-style Pretraining》**

1. **[基础概念题]** VLA-JEPA 将 JEPA（Joint Embedding Predictive Architecture）的核心思想引入 VLA 预训练。请解释 JEPA 与传统的自编码器（AE/VAE）在学习表示上的本质区别，以及 JEPA 如何避免表征坍塌问题。
   - **答案/思考提示**：JEPA 通过预测未来表征而非重建像素，避免了重建任务对细节的过度关注。VLA-JEPA 通过 Target encoder（冻结）与 Student encoder（可学习）的双路径分离，确保预测目标不泄露当前时刻信息，配合 ELBO 目标中的对比正则化防止表征坍塌。

2. **[深度思考题]** VLA-JEPA 的时序因果注意力机制同时满足空间双向和时序因果的约束。请解释这如何在技术上实现，以及这种设计对 VLA 任务为何重要。
   - **答案/思考提示**：空间维度使用双向注意力（理解完整场景），时间维度使用因果 mask（下三角矩阵阻止访问未来）。这确保模型在学习过程中不会泄露未来信息，对应"不能让智能体提前知道答案"的设计哲学。

3. **[优缺点对比题]** 比较 VLA-JEPA 与标准扩散策略（Diffusion Policy）在动作预测上的差异。从预测范式、训练效率、动作分布建模三个角度分析。
   - **答案/思考提示**：扩散策略在像素/动作空间逐步去噪，适合建模多模态动作分布但计算开销大；VLA-JEPA 用流匹配（Flow Matching）预测速度场，1 步采样即可生成动作，效率更高。流匹配提供更好的多模态动作分布同时保持训练稳定性。

4. **[实践应用题]** VLA-JEPA 采用两阶段训练（JEPA 预训练 + 动作头微调）。如果要在新机器人平台上部署，需要如何设计微调策略？考虑数据量有限的情况。
   - **答案/思考提示**：冻结 JEPA 预训练部分，只微调 Flow Matching Action Head；使用 LoRA 或 adapter 减少可训练参数；利用少量本任务演示数据 + 大量预训练数据混合微调；关注动作头过拟合，适当增加 dropout。

**论文：《MINT: Mimic Intent, Not Just Trajectories》**

5. **[基础概念题]** MINT 的核心洞察是"意图（Intent）是低频的，执行（Execution）是高频的"。请解释这个洞察如何通过 SDAT（Spectral Disentangled Action Tokenizer）的技术设计实现。
   - **答案/思考提示**：通过 DCT（离散余弦变换）将动作从时间域转到频率域，S₁（最低频分量）天然对应全局行为形状（意图），S₂~Sₖ（高频分量）对应局部执行细节。尺度感知频域重建损失显式约束每个尺度在频域中承担不同的重建职责。

6. **[深度思考题]** MINT 的 One-Shot 迁移通过"注入 Intent Token"实现，无需梯度更新。这种设计为什么有效？Intent Token 作为任务间迁移的核心载体，其物理含义是什么？
   - **答案/思考提示**：Intent Token 编码了行为的低频结构，是跨任务可迁移的共性信息；Execution Tokens 编码高频细节，依赖环境约束。固定 Intent Token、重建 Execution Tokens，等价于"保持任务目标不变，适应新环境约束"。实验证明 Intent Injection 在 New Layout、New Task、Extended Horizon 三项 OOD 场景平均 77% 成功率 vs 语言微调 17%。

7. **[优缺点对比题]** 比较 MINT 与 CARP、UniVLA 在动作 Tokenization 上的本质差异。为什么说 MINT 的频域解耦是"原则性的"而其他方法是"启发式的"？
   - **答案/思考提示**：CARP/UniVLA 用时间域重建目标，无法显式约束粗粒度表示的语义——即使有多尺度结构，粗粒度 token 仍然可能编码任意信息。MINT 用频域尺度感知损失强制约束：S₁ 必须对应低频（行为意图），S₂~Sₖ 必须对应高频（执行细节）。频率本身就是意图/执行分离的物理对应，无需后验解释。

8. **[实践应用题]** MINT 在真机实验中展现了优秀的跨任务泛化能力。如果要将 Intent Token 注入应用到你的项目中的新型机器人，需要哪些步骤？
   - **答案/思考提示**：① 用新机器人数据训练 SDAT，获得新机器人的动作码书；② 提取少量演示的 Intent Token（S₁）；③ 冻结 S₁，用新场景观测微调 Execution Tokens；④ 评估 Intent 是否跨形态迁移（如机械臂 → 人形手臂）。

**论文：《DVGT: Driving Visual Geometry Transformer》**

9. **[基础概念题]** DVGT 与 VGGT 都是视觉几何 Transformer，但核心区别在于坐标系选择。请解释 DVGT 为什么选择自车坐标系而非相机坐标系，以及这如何实现跨相机泛化。
   - **答案/思考提示**：VGGT 输出在第一帧相机坐标系中，与相机参数耦合；DVGT 输出在自车坐标系中，与相机参数解耦。自车坐标系是所有相机共享的参考系，因此 DVGT 可以适配任意相机数量、焦距、位姿的配置，实现跨场景泛化。

10. **[深度思考题]** DVGT 提出时空分解注意力（Spatial-Temporal Geometry Transformer）包括帧内局部注意、跨视角空间注意、跨帧时序注意。请分析为什么这种分解式设计比全局注意力更适合自动驾驶场景。
    - **答案/思考提示**：全局注意力的计算复杂度是 O(n²)，不适合高分辨率图像 token。分解式设计利用驾驶场景的结构先验——帧内局部注意捕捉单帧像素关系，跨视角注意融合多相机，跨帧注意建模时序一致性。计算效率高，适合实时自动驾驶。

**通用 VLA 领域思考题**

11. **[深度思考题]** 在 VLA 训练中，视觉编码器通常用预训练的 DINOv2/SigLIP 初始化，而动作头需要从头训练。为什么不能端到端联合训练所有模块？联合训练的难点在哪里？
    - **答案/思考提示**：视觉编码器在大规模图文数据上预训练，学到了丰富的视觉先验；动作头需要特定领域的动作数据。联合训练时，梯度可能破坏预训练视觉表示，同时动作数据稀缺导致过拟合。解决思路：两阶段训练、低秩适配（LoRA）、渐进式解冻。

12. **[实践应用题]** 你在项目中如何选择 VLA 的动作表示（末端执行器位置 vs 关节角度 vs 扭矩）？考虑任务类型、数据效率、控制带宽等因素。
    - **答案/思考提示**：末端执行器位置适合精准定位任务（Pick-Place），但需要逆运动学转换；关节角度直接控制，但维数高；扭矩适合力控制任务。实际中常用混合表示或层级策略（高层用位置，低层用扭矩）。数据效率上，端到端从像素学习需要更多数据，有层级先验更数据高效。
