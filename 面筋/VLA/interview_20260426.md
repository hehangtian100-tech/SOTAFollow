## 日期：20260426

### VLA 领域

**论文：《π0.7: A Steerable Generalist Robotic Foundation Model》**

1. **[基础概念题]** π0.7 的 Diverse Context Conditioning（DCC）将传统的单一语言指令扩展为多模态上下文 $\mathcal{C}_t = \{\ell_t, \hat{\ell}_t, g_t, m, c\}$，请解释每个字段的含义及其在训练中的作用。
   - 答案/思考提示：$\ell_t$ 是整体任务语言描述；$\hat{\ell}_t$ 是中间层语义子任务（如"拿起削皮刀"）；$g_t$ 是子目标图像（由世界模型生成）；$m$ 是 Episode 元数据（Quality/Speed/Mistake 三元组）；$c$ 是控制模式（joint/ee）。这些字段让模型能从异构数据中学习而不产生"平均化"效应，是数据多样性与模型性能的"解耦器"。

2. **[深度思考题]** π0.7 能在零样本情况下从双臂机器人迁移到单臂 UR5e 并达到人类专家水平（80% vs 80.6%）。模型并没有复制源机器人的具体动作，而是"自适应发现"了新策略。这一现象说明了什么？
   - 答案/思考提示：这说明模型学习到的是**"意图"（低频策略）**而非"执行"（高频细节）。意图比具体动作更具跨本体迁移性——双臂用协调配合开袋，单臂改用 pick-and-place；双臂倾斜末端执行器，单臂改用垂直抓取。模型学到的是"最终要达成什么状态"而非"具体怎么动"，因此能自适应新本体的物理约束。

3. **[优缺点对比题]** π0.7 的 DCC 策略与传统的 task-specific fine-tuning（如 π*₀.6）相比，核心优势是什么？代价又是什么？
   - 答案/思考提示：优势：一个模型服务所有任务，无需为每个新任务独立训练 RL 微调（π*₀.6 每个任务都要微调），泛化能力更强（尤其组合泛化）。代价：DCC 依赖丰富的上下文信息，在信息不完整时性能可能不如专门微调的模型；此外 Episode Metadata 的标注本身需要成本。

4. **[实践应用题]** π0.7 的数据 scaling 规律显示：有 Metadata 时数据越多（质量越低）性能越强；无 Metadata 时更多低质量数据反而有害。这一规律对机器人数据采集策略有何启示？
   - 答案/思考提示：启示：① 不要盲目追求数据质量而忽视多样性——Metadata 是"解耦器"，能让低质量数据也能被有效利用；② 数据采集时应同时记录 Quality/Speed/Mistake 等元数据，而非仅收集"看起来好的"演示；③ 在有限标注预算下，优先标注 Metadata 而非清理数据——前者对性能提升更显著。

5. **[深度思考题]** π0.7 提出了"语言 Coaching 范式"——用自然语言 step-by-step 指导机器人学习新任务，而无需额外演示。这种范式相比传统数据收集有何潜在优势？
   - 答案/思考提示：优势：① 大幅降低数据采集成本——语言指令比物理演示更容易获取和标注；② 人类可以通过语言传授无法演示的技能（如"力度要轻一些"）；③ Coaching 数据可进一步训练高层语言策略，实现完全自主执行。潜在挑战：自然语言的歧义性、复杂物理操作的表达局限性。

**通用 VLA 领域思考题**

6. **[基础概念题]** VISTA 框架提出用世界模型作为高层规划器生成视觉子目标图像，由 VLA 作为低层执行器跟随执行。与端到端 VLA 相比，这种层次化设计的动机是什么？解决了什么问题？
   - 答案/思考提示：动机：端到端 VLA 难以处理长程任务（14% → 69% 成功率提升）。视觉子目标比纯文本目标提供更丰富的视觉和物理约束，使跨未见物体和新型场景的泛化变得可行。本质上是用"视觉语言"作为中间表示解耦了规划与执行。

7. **[优缺点对比题]** VLA 和 World Model 两种范式在机器人控制任务中各有何优劣？
   - 答案/思考提示：VLA 范式（如 π0.7）：端到端、从头到尾直接学习，但组合泛化能力有限；World Model 范式（如 VISTA）：可解释性强（显式规划），泛化能力好，但推理时需要额外计算生成子目标图像。两者正在融合——Fast-WAM 证明了训练时的 video co-training 才是关键，推理时可以跳过显式未来生成。
