## 日期：20260503

### LLM 领域

**通用思考题（当日无新论文）**

1. [基础概念题] 请解释 Attention 机制中 Q（Query）、K（Key）、V（Value）各自的作用。为什么 attention score 要用 softmax 归一化，而不是直接用点积的结果？
   - **答案/思考提示**：Q 是当前 token 的「查询向量」，K 是所有 token 的「键向量」（用于匹配），V 是所有 token 的「值向量」（用于提取信息）。Attention score Q·K^T 衡量的是「查询与每个键的相关性」，这个相关性作为权重对 V 做加权平均，得到当前 token 的输出。
     - **为什么用 softmax**：①归一化使权重和为1，保证输出是 V 的凸组合，数值稳定；②softmax 的「竞争性」使高相关性的 token 获得更高权重，低相关性接近0，类似于 soft retrieval；③softmax 的平滑性保证梯度稳定，便于训练。如果不用 softmax，点积结果可能数值溢出，且权重无法归一化。

2. [深度思考题] FlashAttention 为什么能将注意力计算的内存复杂度从 O(N²) 降到 O(N)（N=序列长度）？分块（tiling）技术具体是如何实现的？在线 softmax（online softmax）算法在分块计算中起了什么作用？
   - **答案/思考提示**：
     - **O(N²) 内存来源**：标准 Attention 需要存储完整的注意力矩阵 S = QK^T（N×N 个元素）用于反向传播，以及中间结果 P = softmax(S)（也是 N×N）。
     - **分块技术**：将 Q、K、V 分成小块（TILE_SIZE × d），每次只处理一个块，所有块轮流使用 SRAM。关键是不需要同时实例化完整的注意力矩阵。
     - **在线 softmax**：标准 softmax 需要先知道所有元素的 max 和 sum 才能归一化，这要求所有元素都已知。在线 softmax 算法通过维护「局部的 max 和 sum」，在处理每个块后更新，最终得到正确结果。具体来说：m(x) = max(x) 满足 max(x1,x2) = max(max(x1), max(x2))，sum 也有类似的递推性质。这使得分块计算成为可能。
     - **重计算**：反向传播时不需要存储完整的注意力矩阵，只需要存储 softmax 的归一化统计量（m、s），可以根据这些统计量和 Q、K、V 重新计算出注意力梯度。

3. [深度思考题] LoRA（Low-Rank Adaptation）中「低秩」假设的直觉是什么？为什么说在微调 LLM 时，权重矩阵的「增量」ΔW 通常是低秩的？这个假设在什么情况下可能不成立？
   - **答案/思考提示**：
     - **低秩直觉**：LLM 的知识存储在权重矩阵中，但适应新任务时不需要重建整个知识库，只需要「调整」部分关键方向。预训练模型的权重已经包含了大量通用知识，微调的本质是在这个大空间中找到一个新的方向，而不是重建整个空间。
     - **数学解释**：设 W_0 ∈ R^{d×k} 是预训练权重，ΔW 是微调产生的权重变化。对于 LLM 的 transformer 权重，d 和 k 通常很大（数千到数万），但研究表明 ΔW 的奇异值分布是「快速衰减」的——前几个奇异值就能解释大部分方差，这意味着 ΔW 是近似低秩的。通过低秩分解 ΔW = BA（B ∈ R^{d×r}, A ∈ R^{r×k}, r << min(d,k)），可以大大减少参数量。
     - **假设不成立的情况**：①任务与预训练差异极大（如预训练是语言模型，微调是纯图像任务）；②需要记忆大量新知识（知识的维度可能很高）；③对抗性微调场景。实践中，对于大多数 NLU 任务，低秩假设是有效的。

4. [实践应用题] 假设你要在车载嵌入式设备（算力受限，如 NVIDIA Jetson Orin）上部署一个 7B LLM 推理服务，要求：
   - 延迟 < 100ms/token
   - 上下文长度 4K
   - 功耗 < 30W
   
   请说明你会采用哪些优化技术，以及如何权衡模型大小、推理速度和生成质量。
   - **答案/思考提示**：
     - **模型层面**：①量化：INT8 或 INT4 量化，W8A16 或 W4A16 混合量化，在 Orin 上有 TensorCore 支持；②剪枝：结构化或非结构化剪枝减少计算量；③知识蒸馏：用小模型（1-3B）蒸馏大模型知识。
     - **推理层面**：①FlashAttention 的优化（减少内存带宽）；②KV Cache 优化（避免重复计算）；③连续批处理（Continuous Batching）提高 GPU 利用率；④投机解码（Speculative Decoding）：用小模型 draft，大模型 verify。
     - **系统层面**：①模型并行（Tensor Parallelism）将模型分到多个 GPU；②CPU-GPU 协同（部分计算卸载到 CPU）；③图优化（TensorRT、ONNX Runtime）。
     - **权衡**：7B 全精度模型在 Orin 上跑 4K context 几乎不可能满足 100ms 延迟。建议考虑：①用 1.3B-3B 的小模型（如 Phi-3-mini、TinyLlama）；②如果必须用 7B，则降低到 INT8 量化 + 2K context；③质量优先时用较大模型 + 投机解码加速。

5. [优缺点对比题] 对比 RoPE（Rotary Position Embedding）和 ALiBi（Attention with Linear Biases）两种位置编码方案：
   - 它们在处理长上下文（> 4K tokens）和外推（extrapolation，训练时没见过的长度）上的能力有何不同？
   - 为什么 mRoPE（多维 RoPE）特别适合多模态 LLM（VLM）任务？
   - **答案/思考提示**：
     - **RoPE**：通过旋转操作将位置信息融入 Q 和 K，能在 dot-product attention 中自然编码相对位置。优势是理论基础好（三角函数性质保证相对位置可解码），且可以通过 Scaling Laws 平滑扩展到更长上下文。缺点：外推能力有限，需要通过位置插值（PI）或 YaRN 等技术增强。
     - **ALiBi**：在 attention score 上直接加线性偏置 b_|i-j|，不需要显式位置编码。优势是外推能力相对较好（因为偏置是相对位置的线性函数，自然泛化到更远距离）；缺点是偏置形式固定，可能不够灵活。
     - **mRoPE（多维 RoPE）**：将 RoPE 扩展到多维（不仅是 1D 序列位置，还有 2D 空间位置、时间位置等），每个维度用独立的旋转频率。这对于 VLM 特别重要，因为图像/视频中的位置关系是 2D/3D 的，语言描述可能同时涉及空间位置（"左边的苹果"）和时间位置（"先做 X 再做 Y"），mRoPE 可以同时编码这些异构的位置信息。
     - **外推场景**：如果需要处理任意长度的上下文，ALiBi 可能更鲁棒；如果需要在预训练长度内高效计算且有一定外推需求，可以结合 RoPE + 位置插值。
