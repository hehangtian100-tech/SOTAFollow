**《LoRA: Low-Rank Adaptation of Large Language Models》**

1. **[基础概念题]** LoRA 的核心假设是"大语言模型微调过程中权重更新 ΔW 具有低秩性"。请解释为什么这个假设是合理的，以及 ΔW = BA 这种低秩分解形式相比直接优化 ΔW 有什么优势？
   - **答案/思考提示**：虽然模型参数量巨大，但适配下游任务时真正需要调整的自由度很低（内在低秩性）。直接优化 ΔW 需要更新 d×k 个参数，而 BA 分解只需 (d+r)×r + (r+k)×r ≈ 2r(d+k) 个参数，当 r << min(d,k) 时参数量大幅减少。BA 形式还便于做权重合并，推理时无额外开销。

2. **[深度思考题]** LoRA 中 A 矩阵用高斯随机初始化、B 矩阵初始化为 0，这种初始化策略背后的设计意图是什么？为什么不能把 A 和 B 都随机初始化？
   - **答案/思考提示**：保证训练初始阶段 ΔW = 0，模型行为完全等价于未微调状态。如果 B 非零，训练初期 ΔW = BA 会产生非零输出，导致模型行为剧烈震荡，类似于微调初期灾难性遗忘的根源。

3. **[实践应用题]** LoRA 的缩放因子 α/r 有什么作用？如果把 r 从 8 增加到 16，α 需要调整吗？为什么？
   - **答案/思考提示**：BA 的方差与 r 正相关（因为 A 的输出方差 ∝ r），α/r 的设计使 r 变化时整体缩放比例保持不变，输出分布稳定。实验中改变 r 时 α 不需要重新调整，因为 α/2r ≈ α/4r 的缩放效果相近，论文建议 α=16 配合 r=16 使用。

4. **[优缺点对比题]** 对比 LoRA 与 Adapter Layers 两种参数高效微调方法，它们在推理时的计算开销有什么本质差异？为什么很多实际部署场景优先选择 LoRA？
   - **答案/思考提示**：Adapter 在推理时需要额外穿过一个小网络（两曾全连接），增加约 2-3ms 延迟（对长序列尤为明显）。LoRA 可将 BA 权重合并到原始权重 W0 中，推理时完全等价于原模型，零额外计算开销。部署角度：LoRA 每个任务只需保存几 MB 的小矩阵，多任务切换无需改变模型结构。

**《FlashAttention-2: Faster Attention with Better Parallelism》**

5. **[基础概念题]** FlashAttention-1 外层循环遍历 K/V，内层循环遍历 Q，这种循环顺序有什么缺陷？FlashAttention-2 如何通过调换循环顺序解决这个问题？
   - **答案/思考提示**：FA-1 的问题在于：半成品输出 O 在内层循环（遍历 Q 块）期间需要反复从 HBM 读出和写入（每处理一个 Q 块就要读写一次 O）。FA-2 将循环顺序调换：外层遍历 Q 块（Q 在外层循环），内层遍历 K/V 块。这样一块 Q 和它的半成品 O 可以钉在 SRAM 里，K/V 像流水线一样穿过，O 只在算完最终结果后才写回 HBM。

6. **[深度思考题]** FlashAttention-2 能保持与标准注意力"数学等价"，这意味着什么？在实际应用中，这种等价性保证的是什么？
   - **答案/思考提示**：数学等价保证输出结果与标准注意力完全一致（逐元素相等），不是近似。实际应用中的价值：① 精度不会因优化而损失；② 可以直接替换现有系统中的标准注意力，无需重新调参；③ 与混合精度训练、梯度检查等技术完全兼容。这使得 FA 成为少数能"免费"获得加速的优化（不牺牲精度）。

**《Kimi Attention Residuals》**

7. **[深度思考题]** Kimi Attention Residual 将"当前层只能接收上一层输出"的固定残差路径改为"当前层可从所有历史层检索最相关信息"。这与标准残差连接在信息流角度有什么本质区别？门控向量 g_l 在其中扮演什么角色？
   - **答案/思考提示**：标准残差信息只能沿层级链一层层传递（O(L) 路径长度），深层要访问浅层信息必须"穿越"所有中间层，中间层的信息可能被稀释或覆盖。Attention Residual 让每层通过 cross-attention 直接从所有历史层检索（类似 skip-connection 的极端形式）。门控 g_l 决定每个 token 自适应地选择"更依赖跨层历史信息"还是"更依赖本地 transformation"，比固定残差比例（PreNorm 恒等映射）更灵活。

8. **[实践应用题]** 论文强调"Infra 决定结构创新上限"，跨层 attention 带来了哪些工程挑战？为什么这些挑战让普通团队难以复现？
   - **答案/思考提示**：① 显存挑战：标准 Transformer KV Cache 只需缓存最后一层，Attention Residual 需要缓存所有历史层的 Key/Value，100层×1M上下文窗口显存膨胀约100倍；② 计算挑战：每层对所有历史层做 attention，O(L²) 级别增长；③ 需要 CUDA/HBM 层面的特殊内存管理和定制化 kernel 实现。这需要顶尖 Infra 团队和硬件层面的深度优化，普通团队难以复现。

**《PETR: Position Embedding Transformation for Multi-View 3D Object Detection》**

9. **[基础概念题]** PETR 通过 3D 位置嵌入变换将多视图 2D 特征转化为 3D 位置感知特征，请描述这个变换的完整流程（从坐标生成到特征融合），以及它与 BEV 方法的核心区别。
   - **答案/思考提示**：流程：① 相机锥台空间离散为 3D 网格 → ② 通过逆投影矩阵转为 3D 世界坐标 → ③ 归一化到 [0,1] → ④ MLP 生成 3D PE → ⑤ 与 2D 特征逐元素相加。与 BEV 的核心区别：PETR 不显式生成 BEV 表示，避免了 Z 轴误差和 2D→3D 投影的信息损失；目标查询直接在 3D 空间与 3D 位置感知特征交互。

10. **[优缺点对比题]** 对比 DETR3D（3D query 投影回 2D 采样）和 PETR（隐式 3D 位置编码），两种方法各自的优缺点是什么？PETR 的方案在工程实现上有什么潜在代价？
    - **答案/思考提示**：DETR3D 优点：无需生成完整的 3D/ BEV 表示；缺点：投影误差、采样不全面、多视角特征融合复杂。PETR 优点：端到端、无投影采样、跨视图空间关联自然；缺点：3D 坐标离散化（D=64 depth bins）存在量化误差，MLP 学习 3D 位置编码的表达能力可能受限。PETR 的代价：深度维度 D=64 使得特征从 H×W 扩展到 H×W×D，计算量和显存显著增加（对于 1056×384 输入，H×W×D = 24×48×64）。

**《ZeRO: Memory Optimizations Toward Training Trillion Parameter Models》**

11. **[基础概念题]** 混合精度训练中，模型参数、梯度、优化器状态三者的显存占用比例大约是多少？ZeRO 各阶段（-1/-2/-3）分别优化了哪部分的显存？
    - **答案/思考提示**：混合精度下：参数 FP16（2P）+ 梯度 FP16（2P）+ 优化器状态 FP32（12P），总计 16P 字节。ZeRO-1：只分区优化器状态，减少 4×（主要是 12P → 3P）；ZeRO-2：分区优化器状态 + 梯度，减少 8×；ZeRO-3：分区所有状态（参数+梯度+优化器），减少 Nd 倍（Nd 为数据并行度）。

12. **[深度思考题]** ZeRO-3 在前向传播时需要 All-Gather 收集完整参数才能计算，但计算完成后立即释放非本地参数。这种"用时获取、不用即释放"的策略如何实现？在实际训练中，All-Gather 通信能否被计算遮挡？
    - **答案/思考提示**：通过分片管理：每个 GPU 只存储 1/Nd 的参数分片，前向计算前用 All-Gather 从所有 GPU 收集完整参数（Nd-1 个 GPU 的分片），计算完成后释放。通信可与计算重叠：当模型层数较多时，前向传播的 All-Gather 可与下一层的计算并行执行，ZeRO 论文显示通信开销可被计算完全遮挡，训练效率接近零开销。
