**论文1：《PETR V2：统一3D感知框架（检测/分割/车道线）》**

1. **[基础概念题]** PETR V2 提出了"3D 坐标时序对齐"（3D Coordinates Alignment）来实现时序建模。请解释：为什么要在 3D 空间做对齐而不是直接在 2D 特征空间对齐？这样做有什么优势？
   - **答案/思考提示**：因为相机之间外参不同，直接在2D特征空间对齐会受到相机位姿差异的影响。在3D空间做对齐后，3D坐标本身就包含了视角信息，再投影到统一坐标系可以实现视角无关的时序融合。同时3D PE天然带有几何约束，对相机外参噪声更鲁棒。

2. **[深度思考题]** PETR V2 的 FPE（Feature-guided Position Encoder）将 3D PE 从"数据无关"变为"数据驱动"。请分析：这种设计为什么能提升模型对相机外参噪声的鲁棒性？这背后的物理直觉是什么？
   - **答案/思考提示**：原3D PE只依赖几何坐标，与图像内容无关。FPE通过图像特征加权，让3D PE的生成受到图像内容的引导。当相机外参有噪声时，几何计算的3D位置可能不准，但图像特征（边缘、纹理等）仍然可靠——FPE利用这种互补性提升鲁棒性。

3. **[实践应用题]** PETR V2 使用稀疏的任务专属查询（Detection Query / Segmentation Query / Lane Query）而非稠密的 BEV Query。请分析这种设计在计算效率和任务专用性上的权衡。在什么场景下你可能仍然倾向于使用稠密 BEV Query？
   - **答案/思考提示**：稀疏查询计算量小、每个任务可独立优化，但可能错过跨任务的信息共享。稠密BEV Query适合需要全局上下文融合的任务（如密集预测的地图分割）。当多任务需要强交互时，可考虑混合架构。

4. **[优缺点对比题]** 对比 PETR V2（DETR-style）和 BEVFormer（BEV Query-based）在时序建模上的方法差异。哪种方法在相机外参有噪声时更鲁棒？为什么？
   - **答案/思考提示**：BEVFormer通过 deformable attention 在BEV空间做时序融合，受益于BEV表示的视角一致性。PETR V2在3D坐标层面做对齐，物理意义更直接。PETR V2的FPE对噪声更鲁棒（参考消融实验），因为图像特征引导可以补偿几何误差。

**论文2：《SwiftVGGT：稀疏3D大规模场景重建》**

1. **[基础概念题]** SwiftVGGT 的核心创新是去除了外部 VPR（Visual Place Recognition）模型来实现回环检测。请解释：传统方法为什么需要 VPR？SwiftVGGT 如何用特征相似性替代它？
   - **答案/思考提示**：VPR用于在大规模场景中快速定位"曾经来过的地方"（回环检测），避免误差累积。SwiftVGGT观察到：当两帧存在重叠区域时，特征图在对应位置会产生高相似性响应。直接利用特征相似度匹配可以检测回环，无需额外的检索模型。

2. **[深度思考题]** SwiftVGGT 用 Sim(3)-SVD 块对齐替代了传统的 IRLS（Iteratively Reweighted Least Squares）优化。Sim(3) 相比 SE(3) 多了什么？为什么尺度因子 s 对点云配准很重要？
   - **答案/思考提示**：Sim(3)比SE(3)多了尺度因子。实际3D重建中，不同chunk的深度尺度可能不一致（尤其是单目估计），加入尺度因子可以校正这种系统性尺度偏差，使不同chunk的点云能够正确对齐到统一坐标系。

3. **[实践应用题]** SwiftVGGT 声称是"Training-Free"的——无需任何网络训练。请分析：这种方法的泛化能力来自哪里？它与端到端学习方法相比有什么固有的局限性？
   - **答案/思考提示**：泛化能力来自预训练特征（DINOv2）——在ImageNet上学习到的通用视觉表征。局限性：①依赖预训练特征质量 ②对低纹理区域可能失效 ③几何方法无法利用任务特定信号进行优化。适合作为初始化或通用工具，而非端到端可学习的方案。

4. **[优缺点对比题]** 对比 SwiftVGGT 与 VGGT（论文中提到的基线）在精度、速度和适用场景上的差异。SwiftVGGT 3倍加速的背后牺牲了什么？
   - **答案/思考提示**：SwiftVGGT牺牲了约1-3%的精度换取3倍加速（从156s降到52s on ScanNet）。精度损失主要来自①去除了VPR的精确回环检测 ②Sim(3)-SVD替代IRLS迭代优化。对于精度要求极高的专业测绘场景，VGGT仍更合适；对于需要实时重建的机器人导航，SwiftVGGT更实用。

**论文3：《VGGT：Visual Geometry Grounded Transformer》**

1. **[基础概念题]** VGGT 是一个多任务Transformer，同时预测相机参数、深度图、点云和跟踪特征。请解释：为什么"预测相互关联的3D属性"（深度+相机→点云）反而能提升整体准确率，而不是造成任务间的干扰？
   - **答案/思考提示**：因为相机几何本质上约束了深度和点云的数学关系——点云 = 深度 × 相机射线方向。联合学习让模型能够利用这些几何约束作为归纳偏置，而非盲目拟合。当相机估计更准时，深度估计也会受益，反之亦然，形成正向循环。

2. **[深度思考题]** VGGT 的消融实验显示"交替注意力"（帧内 + 全局）比纯全局注意力效果更好。请从"局部匹配 vs 全局一致性"的角度解释这一现象。
   - **答案/思考提示**：帧内注意力让模型先在单帧内建立像素级匹配（几何局部性），全局注意力再跨帧建立长程一致性。这符合多视角几何的物理过程——先建立帧内特征对应，再融合多视角约束。纯全局注意力缺乏这种层次化结构。

3. **[实践应用题]** VGGT 提到可以作为"通用3D特征骨干"迁移到下游任务（如点跟踪、新视角合成）。请设计一个具体的应用案例：如何用预训练VGGT特征来提升点跟踪性能？
   - **答案/思考提示**：将VGGT的tracking head输出的特征作为点跟踪的描述子。给定第一帧中某个3D点的特征，在后续帧中搜索特征最接近的位置，结合相机参数做几何验证。由于VGGT特征包含了3D几何信息，对遮挡和外观变化的鲁棒性会比纯2D特征更强。

4. **[优缺点对比题]** 对比 DUSt3R/MASt3R 与 VGGT 在以下维度：输入尺度、多任务统一性、后处理依赖、推理速度。VGGT的前馈范式是否是3D重建的终极答案？
   - **答案/思考提示**：VGGT的前馈推理证明了"学习+预测"可以替代部分传统几何优化，但BA后处理仍能进一步提升精度。VGGT并非终极答案——它更适合作为高质量初始化，配合可微优化做联合精化。两种范式结合（learned initialization + optimization refinement）是更现实的路线。

**论文4：《World Model / VLA 自回归框架详解》**

1. **[基础概念题]** 在 VLA 框架中，Action Token Masking 有三种常见方案：Action as Output（RT-2）、Action as Input（OpenVLA）、Action Chunking（ACT）。请分别解释它们的掩码设计，并分析哪种方案最适合机器人实时控制？为什么？
   - **答案/思考提示**：RT-2把动作只作输出防止视觉信息被动作干扰；OpenVLA让动作token参与注意力实现双向交互；ACT一次性预测整个动作序列。实时控制需要低延迟→Action as Output虽然简单但推理最快；ACT的chunking可以在速度和连贯性间取得平衡。

2. **[深度思考题]** 文档详细对比了 Teacher Forcing 训练与自回归推理的差异（Exposure Bias）。请解释：为什么这种训练-推理不一致在 VLA 场景中比 NLP 中更危险？有什么缓解方法？
   - **答案/思考提示**：NLP中token预测错误通常只影响局部语义；VLA中动作预测错误会导致物理状态偏离，错误会级联放大（机器人可能摔倒或碰撞）。缓解方法：① Scheduled Sampling渐进引入模型自己的预测 ② Diffusion-based方法建模动作分布而非点估计 ③ 使用世界模型做内部仿真验证

3. **[实践应用题]** 假设你需要实现一个基于 Uni-World VLA 的"交错生成"机制来同时预测视频帧和动作。请设计这个机制的核心伪代码，说明如何构建交错掩码以实现"看历史帧→生成动作→预测下一帧→再生成动作"的循环。
   - **答案/思考提示**：关键是在每个World Model Block内构建因果掩码：① 历史帧对当前帧可见，但未来帧不可见 ② 动作只能看历史帧+当前块内已生成的动作 ③ 当前帧的预测不能看当前块内的动作（避免循环依赖）→ 交替进行，当前帧条件于动作，动作条件于预测帧

4. **[优缺点对比题]** 对比纯 Diffusion 动作生成与自回归（AR）动作生成的优劣。在什么场景下你会选择 Diffusion 而非 AR？反之呢？
   - **答案/思考提示**：Diffusion优点：多模态动作分布建模好、推理时可做 classifier-free guidance引导。AR优点：与LLM架构统一、可复用预训练权重、推理速度快（无需多步去噪）。高精度任务（如手术机器人）→ Diffusion；需要语言引导的开放域任务 → AR；实时控制 → AR或单步Diffusion

**论文5：《VQVAE视觉Tokenizer详解》**

1. **[基础概念题]** VQVAE 的量化操作（quantize）本质上是不可导的。请解释：Straight-Through Estimator（STE）如何绕过这个问题？这种近似对训练有什么潜在影响？
   - **答案/思考提示**：STE前向传播用量化后的码字（不可导），反向传播时将梯度直接复制给Encoder输出（绕过量化操作）。影响：① Encoder收到的梯度是近似的（假设量化点局部线性） ② 可能导致码本更新不及时（梯度被旁路）→ 使用EMA更新码本可以缓解

2. **[深度思考题]** VQVAE 的损失函数包含三项：重构损失、VQ损失、承诺损失。请解释承诺损失（Commitment Loss）的作用：为什么需要单独约束 Encoder 输出不能偏离码本太远？
   - **答案/思考提示**：没有承诺损失，Encoder可能将同一图像块编码到不同位置（"码本漂移"），导致码字使用不均匀、训练不稳定。承诺损失让Encoder"承诺"自己的输出会靠近某个码字，保持量化过程的稳定性。

3. **[实践应用题]** 在 World Model 中使用 VQVAE 作为视觉Tokenizer时，码本大小 K 是一个关键超参数。请分析：K 太大或太小分别会导致什么问题？如何选择合适的 K 值？
   - **答案/思考提示**：K太小→码本容量不足，相似视觉模式被强制映射到同一码字，表征能力受限（码本崩溃风险）。K太大→接近连续表征，STE梯度估计误差增大，训练困难。经验法则：K的选择应与压缩比匹配，常见8192~65536，需通过重构质量和码本使用率来验证。

4. **[优缺点对比题]** 对比 VQVAE 与 VAE/β-VAE 作为 World Model 视觉Tokenizer的优劣。为什么当前主流 World Model（如 Dreamer、π0）大多选择 VQVAE 而非连续表征？
   - **答案/思考提示**：VAE连续表征的问题：① 难以与离散token的语言模型结合 ② 隐空间不稀疏 ③ 不能直接做离散化的"想象力生成"。VQVAE的优势：离散token可与LLM统一建模（next token prediction）、压缩率高（256×256→16×16，768:1）、码本提供"视觉词表"的语义聚合能力。

**论文6：《RoPE、3DPE与mRoPE技术详解》**

1. **[基础概念题]** RoPE（旋转位置编码）的核心洞察是"通过旋转编码绝对位置，attention score自然变成相对位置"。请推导：为什么旋转角度差只与相对位置有关？这一性质如何保证外推能力？
   - **答案/思考提示**：(Rot(mθ)q)·(Rot(nθ)k) = q·Rot((m-n)θ)k，只依赖相对角度(m-n)。外推能力来自：训练时见过位置0~L，推理时可以外推到L+k——因为旋转角度的相对差值计算不需要上界。

2. **[深度思考题]** ALiBi 使用线性偏置实现远程衰减，RoPE 使用旋转矩阵实现相对位置编码。请分析：这两种方法在数学形式和工程实现上有什么本质差异？为什么 RoPE 成为 LLM 的主流选择？
   - **答案/思考提示**：ALiBi在attention score上直接加减距离项，与token内容无交互；RoPE通过旋转矩阵乘积实现，位置信息融入Q/K向量，可参与后续计算。RoPE优势：① 无需额外参数 ② 与flash attention等优化更兼容 ③ 表征能力更强（位置信息可二次计算）

3. **[实践应用题]** 假设你需要将 RoPE 从1D扩展到3D（用于视频或多视角图像），请设计一个3D RoPE的方案：如何将 x、y、t 三个维度的旋转组合起来？每维的旋转基频率如何设置？
   - **答案/思考提示**：3D RoPE可采用分治策略：分别在(x,y,t)三个子空间做1D RoPE，然后拼接或逐维度应用。频率设置：通常空间维度频率高于时间维度（空间细节更密集），可用 base_x=10000^(2i/(3d)) 等比例分配。

4. **[优缺点对比题]** 对比 Learned PE、Sinusoidal PE、RoPE 在外推性、相对位置感知、参数量三个维度上的表现。为什么现代LLM几乎都选择 RoPE？
   - **答案/思考提示**：Learned PE外推差（超出训练长度无法处理）、Sinusoidal外推好但无参数学习、RoPE两者兼顾且零参数。RoPE成为主流因为：① 与GPT等decoder-only架构天然适配 ② 可与Flash Attention等高效实现结合 ③ 理论优雅且工程成熟（LLaMA/Qwen/Gemma均采用）

**论文7：《大模型基础知识Roadmap》**

1. **[基础概念题]** RLHF 和 DPO 都是用于对齐大模型的方法，但核心思想不同。请从损失函数角度解释：为什么 DPO 可以绕过 Reward Model 直接优化偏好？DPO 的隐含假设是什么？
   - **答案/思考提示**：DPO将reward转化为策略的隐式表达，损失直接最大化偏好数据似然而非先学reward再PPO。隐含假设：参考策略（reference model）是准确的偏好估计——当参考模型质量高时DPO有效，否则可能学偏。

2. **[深度思考题]** Scaling Laws 指出模型参数N、训练数据D、计算量C之间存在幂律关系。Chinchilla 批评GPT-3只优化N而忽视了D，提出N和D应同比例 scaling。请分析：在有限计算预算下，scaling N vs D 哪个对模型最终效果影响更大？有什么例外情况？
   - **答案/思考提示**：通用场景下两者同等重要（Chinchilla最优）。例外：① 当数据质量极高时，增加N可能优于增加D（高质量数据边际收益递减慢） ② 对于特定领域任务，可能需要更大的N来吸收知识 ③ 推理效率优先时，可能选择更小的N（蒸馏）

3. **[实践应用题]** 假设你需要将一个70B大模型部署到消费级GPU（24GB显存）上。请从量化、剪枝、知识蒸馏三个维度分别选择最合适的技术方案，并说明理由。
   - **答案/思考提示**：量化→4-bit GPTQ/AWQ（最常用，70B可压到~40GB）；剪枝→Wanda或SparseGPT（无需重训练）；知识蒸馏→用大模型蒸馏小模型（如LLaMA-7B）。组合方案：先剪枝再量化可进一步压缩。最终大小受推理延迟要求约束。

4. **[优缺点对比题]** 对比 MoE（Mixture of Experts）与 Dense Transformer 在训练稳定性、推理效率、显存占用三个维度上的差异。为什么当前超大模型（如DeepSeek-V2、Grok-1）普遍采用MoE架构？
   - **答案/思考提示**：MoE训练更不稳定（负载均衡问题）但推理效率高（只激活部分参数）；Dense训练稳定但推理成本高。超大模型选MoE核心原因：参数量大但推理只需激活一小部分，在保持模型能力的同时大幅降低推理计算成本（MoE的"参数量"≠"激活量"）。
